HTML

  • HTML elements are used to create the structure of a webpage, provide content and basic framework for webpages
  • The head element contains information about the webpage not visible to the user
  • The body element represents the visible content shown to the user
  • A semantic element describes its content and the role it plays on the webpage; a non-semantic element can be used to solve structural problems
  • The h1 to h6 tags represent headers in different sizes
  • The ul tag represents an unordered or unnumbered list
  • The li tag represents each list item
  • The img tag contains info on how to display an image and the src attribute defines the location of the image
  • The br tag creates a line of empty space
  • The a href tag is used to create hyperlinks
  • The section tag can be used to break up content on a page

CSS

  • To style an element directly in the HTML file, we use inline CSS, which can be useful for quick and permanent changes but are less flexible than using an internal or external style sheet
  • Internal style sheets are embedded directly in the HTML file using a style element within the head element; these rules only apply to that page
  • The most common and preferred way to style a webpage is by using an external CSS style sheet
  • An external style sheet allows developers to keep all their CSS rules in a separate file, making design changes easier
  • Use a link element placed in the HTML's head element to link to an external CSS
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Assign colors in CSS with semantic references such as gray or blue, or by its hexadecimal code, or hex #
  • Use class selectors to designate which elements in your HTML will contain those property assignments

Git

  • A repository or "repo" is a folder for your project that contains all of the files and revision history
  • Creating a new branch allows you to make changes that won't affect the live site
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to that branch
  • Create a Pull Request to ask for changes on a branch to merge into another branch
  • Use git commit -m to add a description to the changes you're making to your branch

JavaScript

  • JavaScript adds logic and functionality to a webpage or application
  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • In JS, code runs from the first statement at the top of the page to the last statement at the bottom of the page
  • "IF" statements allow us to interrupt the control flow, which allows us to build apps that respond to a user's actions
  • Use an array to hold a group of related data in a single variable
  • Indexes assign a number to each item in an array, starting with 0
